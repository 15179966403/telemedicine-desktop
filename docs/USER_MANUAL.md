## 系统介绍

### 产品概述

互联网医院桌面端是专为医生设计的办公应用，提供高效的患者管理和在线问诊服务。应用充分利用桌面端多窗口管理的优势，让医生能够同时处理多个患者的问诊请求。

### 主要功能

- **患者管理**：查看和管理患者信息、病历档案
- **在线问诊**：实时图文问诊、语音消息、文件传输
- **多窗口支持**：同时处理多个患者问诊
- **离线功能**：网络不稳定时仍可查看基本信息
- **数据安全**：加密存储、自动锁屏、操作日志

### 系统要求

- **操作系统**：
  - Windows 10 或更高版本
  - macOS 10.13 或更高版本
  - Ubuntu 20.04 或更高版本
- **硬件要求**：
  - 处理器：双核 2.0 GHz 或更高
  - 内存：4 GB RAM 或更高
  - 硬盘：500 MB 可用空间
  - 网络：稳定的互联网连接

## 安装指南

### Windows 安装

1. 下载安装包（.exe 或 .msi 文件）
2. 双击安装包运行安装程序
3. 按照安装向导提示完成安装
4. 安装完成后，从开始菜单或桌面快捷方式启动应用

### macOS 安装

1. 下载 DMG 镜像文件
2. 双击打开 DMG 文件
3. 将应用图标拖拽到"应用程序"文件夹
4. 从启动台或应用程序文件夹启动应用
5. 首次启动时，可能需要在"系统偏好设置 > 安全性与隐私"中允许运行

### Linux 安装

#### 使用 DEB 包（Debian/Ubuntu）

```bash
sudo dpkg -i telemedicine-desktop_1.0.0_amd64.deb
sudo apt-get install -f  # 安装依赖
```

#### 使用 AppImage

```bash
chmod +x telemedicine-desktop_1.0.0_amd64.AppImage
./telemedicine-desktop_1.0.0_amd64.AppImage
```

## 快速开始

### 首次登录

1. **启动应用**：双击桌面图标或从应用列表启动

2. **选择登录方式**：
   - 账号密码登录
   - 短信验证码登录

3. **输入凭证**：
   - 账号密码：输入医生账号和密码
   - 短信验证码：输入手机号，点击"获取验证码"，输入收到的验证码

4. **实名认证**（首次登录）：
   - 上传医师资格证
   - 填写执业信息
   - 等待审核通过

5. **进入工作台**：登录成功后进入主工作台界面

## 功能详解

### 1. 患者管理

#### 查看患者列表

1. 点击左侧导航栏"患者"
2. 患者列表显示所有关联的患者信息
3. 支持按以下条件筛选：
   - 姓名
   - 病情
   - 就诊时间
   - 标签

#### 搜索患者

1. 在搜索框输入关键词
2. 支持搜索：
   - 患者姓名
   - 手机号
   - 病情描述
3. 实时显示搜索结果

#### 查看患者详情

1. 点击患者卡片
2. 查看详细信息：
   - 基本信息：姓名、年龄、性别、联系方式
   - 病历档案：历史病历、诊断记录
   - 检查报告：影像资料、化验单
   - 用药记录：处方历史、用药情况

#### 管理患者标签

1. 在患者详情页点击"添加标签"
2. 输入标签名称（如：高血压、糖尿病）
3. 点击保存
4. 可在患者列表按标签筛选

### 2. 在线问诊

#### 接收问诊订单

1. 新订单会在"待接诊"列表显示
2. 系统会发送桌面通知
3. 点击订单查看详情：
   - 患者基本信息
   - 症状描述
   - 上传的图片/报告

#### 开始问诊

1. 点击"开始问诊"按钮
2. 系统打开新的问诊窗口
3. 问诊窗口包含：
   - 聊天区域
   - 患者信息面板
   - 病历查看区域

#### 图文问诊

**发送文字消息**：

1. 在输入框输入消息
2. 按 Enter 发送（Shift+Enter 换行）

**发送图片**：

1. 点击图片图标
2. 选择要发送的图片
3. 支持格式：JPG、PNG、GIF
4. 单张图片最大 10MB

**发送语音**：

1. 点击语音图标
2. 按住录音按钮说话
3. 松开发送，上滑取消

**发送文件**：

1. 点击文件图标
2. 选择要发送的文件
3. 支持格式：PDF、DOC、DOCX、XLS、XLSX
4. 单个文件最大 20MB

#### 使用医嘱模板

1. 点击"常用医嘱"按钮
2. 选择合适的模板
3. 可编辑模板内容
4. 点击发送

#### 查看病史

1. 点击右侧"病历"标签
2. 查看患者既往病历
3. 可按时间筛选
4. 点击病历查看详情

#### 完成问诊

1. 问诊结束后点击"完成问诊"
2. 填写诊断结果
3. 开具处方（如需要）
4. 提交完成

### 3. 多窗口管理

#### 打开新窗口

1. 在待接诊列表点击"新窗口打开"
2. 或在患者详情页点击"新窗口查看"
3. 系统会创建独立的问诊窗口

#### 窗口切换

1. 使用任务栏切换窗口
2. 或使用快捷键：
   - Windows/Linux：Alt + Tab
   - macOS：Cmd + Tab

#### 窗口管理

- **最小化**：点击窗口标题栏的最小化按钮
- **最大化**：点击窗口标题栏的最大化按钮
- **关闭**：点击窗口标题栏的关闭按钮
  - 关闭前会提示保存未完成的操作

#### 窗口限制

- 系统最多同时打开 10 个问诊窗口
- 超过限制时会提示关闭其他窗口
- 建议根据电脑性能合理控制窗口数量

### 4. 消息中心

#### 查看消息

1. 点击左侧导航栏"消息"
2. 查看所有会话列表
3. 未读消息会显示红点提示

#### 消息状态

- **发送中**：消息正在发送
- **已发送**：消息已发送到服务器
- **已送达**：患者已收到消息
- **失败**：消息发送失败，点击重试

#### 历史消息

1. 在聊天窗口向上滚动
2. 系统自动加载历史消息
3. 支持搜索历史消息

### 5. 离线功能

#### 离线查看

网络断开时：

- 可查看已缓存的患者基本信息
- 可查看历史聊天记录
- 无法发送新消息
- 无法接收新订单

#### 离线消息队列

1. 离线时发送的消息会加入队列
2. 网络恢复后自动发送
3. 发送失败的消息可手动重试

#### 网络状态提示

- 顶部会显示网络状态
- 离线时显示"离线模式"提示
- 网络恢复时显示"已连接"

### 6. 安全功能

#### 自动锁屏

- 5 分钟无操作自动锁屏
- 锁屏后需要重新输入密码
- 可在设置中调整锁屏时间

#### 数据加密

- 所有数据传输使用 HTTPS 加密
- 敏感数据本地加密存储
- 应用关闭时清除内存数据

#### 操作日志

- 系统记录所有敏感操作
- 包括：登录、查看患者信息、发送消息
- 日志用于审计和安全追溯

### 7. 设置

#### 个人信息

- 查看和编辑个人资料
- 修改密码
- 绑定手机号

#### 通知设置

- 开启/关闭桌面通知
- 设置通知声音
- 设置通知显示内容

#### 隐私设置

- 自动锁屏时间
- 数据保留策略
- 清除缓存数据

- 查看应用版本
- 检查更新
- 用户协议和隐私政策

## 常见问题

### Q1: 忘记密码怎么办？

A: 在登录页面点击"忘记密码"，通过手机验证码重置密码。

### Q2: 如何同时处理多个患者？

A: 在待接诊列表中，对每个患者点击"新窗口打开"，即可在独立窗口中处理。

### Q3: 消息发送失败怎么办？

A:

1. 检查网络连接
2. 点击失败消息旁的重试按钮
3. 如果持续失败，请联系技术支持

### Q4: 如何查看患者的历史病历？

A: 在问诊窗口右侧点击"病历"标签，即可查看患者的历史病历记录。

### Q5: 应用可以在多台电脑上使用吗？

A: 可以，但同一账号同时只能在一台设备上登录。新设备登录会使旧设备下线。

### Q6: 如何更新应用？

A: 应用会自动检查更新。有新版本时会弹出提示，点击"立即更新"即可。

### Q7: 离线时能做什么？

A: 离线时可以查看已缓存的患者信息和历史消息，但无法发送新消息或接收新订单。

### Q8: 如何保护患者隐私？

A:

- 不要在公共场所使用应用
- 离开时及时锁屏
- 不要截图或拍照患者信息
- 定期修改密码

## 故障排查

### 无法登录

**可能原因**：

- 账号或密码错误
- 网络连接问题
- 账号被锁定

**解决方法**：

1. 检查账号密码是否正确
2. 检查网络连接
3. 尝试使用短信验证码登录
4. 联系管理员解锁账号

### 消息发送失败

**可能原因**：

- 网络不稳定
- 文件过大
- 服务器繁忙

**解决方法**：

1. 检查网络连接
2. 压缩大文件后重试
3. 等待片刻后重试
4. 联系技术支持

### 应用卡顿

**可能原因**：

- 打开窗口过多
- 内存不足
- 缓存过多

**解决方法**：

1. 关闭不需要的窗口
2. 重启应用
3. 清理缓存数据
4. 升级电脑硬件

### 无法打开新窗口

**可能原因**：

- 已达到窗口数量限制
- 系统资源不足

**解决方法**：

1. 关闭一些已打开的窗口
2. 重启应用
3. 检查系统资源使用情况

### 更新失败

**可能原因**：

- 网络连接问题
- 磁盘空间不足
- 权限不足

**解决方法**：

1. 检查网络连接
2. 清理磁盘空间
3. 以管理员身份运行
4. 手动下载安装包更新
